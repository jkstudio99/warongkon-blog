ถ้าจะเลือกข่าว AI ระดับโลกที่มีผลต่อทั้งฝั่งโมเดล ฝั่งคลาวด์ และฝั่งลูกค้าองค์กรพร้อมกัน ข่าวที่ชัดที่สุดในปลายเดือนนี้คือการที่ **Microsoft และ OpenAI ประกาศแก้ไขข้อตกลงความร่วมมืออีกครั้งในวันที่ 27 เมษายน 2026**

ทั้งสองบริษัทโพสต์ข้อความในวันเดียวกันและใช้สาระหลักตรงกันเกือบทั้งหมด นั่นทำให้ข่าวนี้มีน้ำหนักมากกว่าการ “ปล่อยข่าวลือผ่านสื่อ” เพราะเป็นการยืนยันจากคู่สัญญาโดยตรง

ประเด็นสำคัญมี 5 ข้อ:

1. Microsoft ยังเป็น **primary cloud partner** ของ OpenAI
2. แต่สิทธิ์ของ Microsoft ต่อ IP ของ OpenAI จะเป็นแบบ **non-exclusive**
3. OpenAI สามารถให้บริการผลิตภัณฑ์ของตัวเองบน **cloud รายอื่นได้**
4. Microsoft จะ **ไม่ต้องจ่าย revenue share ให้ OpenAI** อีกต่อไป
5. OpenAI ยังต้องจ่าย revenue share ให้ Microsoft ต่อถึง **ปี 2030** แต่มี **cap** และไม่ผูกกับ milestone ทางเทคโนโลยีแบบเดิม

จุดนี้สำคัญมาก เพราะมันไม่ได้เป็นแค่การแก้คำในสัญญา แต่คือการยอมรับตรง ๆ ว่าโลก AI ปี 2026 ไปไกลเกินกว่าจะล็อกทุกอย่างไว้กับโครงสร้าง partnership แบบปี 2023

## สิ่งที่เกิดขึ้นจริงในวันที่ 27 เมษายน 2026

โพสต์ของ Microsoft ลงวันที่ **27 เมษายน 2026** ระบุว่าข้อตกลงฉบับแก้ไขถูกทำขึ้นเพื่อเพิ่ม “flexibility” และ “certainty” ให้ทั้งสองฝ่าย

ขณะเดียวกัน OpenAI ก็เผยแพร่ข้อความเวอร์ชันของตัวเองในวันเดียวกัน โดยย้ำสารเดียวกันว่า Microsoft ยังเป็นพาร์ตเนอร์หลักด้านคลาวด์ แต่ OpenAI มีทางเลือกให้บริการสินค้าใน **cloud อื่น** ได้แล้ว หาก Microsoft ไม่รองรับความสามารถที่ต้องใช้ หรือในกรณีที่ OpenAI ต้องการขยายการให้บริการในวงกว้างมากขึ้น

นี่คือจุดเปลี่ยนเชิงโครงสร้าง เพราะในช่วงก่อนหน้า narrative หลักของตลาดคือ:

- Azure คือบ้านหลักของ OpenAI
- Microsoft ถือสิทธิ์เชิงพาณิชย์ที่แข็งแรงมาก
- ความสัมพันธ์นี้เกือบจะถูกมองว่าเป็น “quasi-exclusive alliance”

แต่หลังประกาศวันที่ 27 เมษายน ภาพนั้นเปลี่ยนเป็น:

- Azure ยังสำคัญ
- Microsoft ยังได้สิทธิ์ระยะยาวถึง **2032**
- แต่ OpenAI ไม่ได้ถูกผูกไว้กับ ecosystem เดียวอีกต่อไป

## ทำไมคำว่า non-exclusive ถึงแรงกว่าที่ดู

ถ้ามองผิวเผิน หลายคนอาจคิดว่า “ก็ยังเป็น partner กันเหมือนเดิม” ซึ่งก็จริงแค่ครึ่งเดียว

คำว่า **non-exclusive** สำหรับสิทธิ์ของ Microsoft ต่อ IP ของ OpenAI หมายความว่า Microsoft ยังได้ประโยชน์จากโมเดลและผลิตภัณฑ์ของ OpenAI ต่อ แต่ไม่ได้ผูกขาดแบบเดิม

ผลกระทบที่ตามมาคือ:

- ลูกค้าองค์กรอาจเห็น OpenAI ไปอยู่บนหลาย cloud มากขึ้น
- OpenAI มี leverage ในการคุยกับผู้ให้บริการโครงสร้างพื้นฐานรายอื่น
- ตลาด cloud AI จะกลับมาแข่งขันกันด้วย **ราคา ความพร้อมของ capacity และ integration** มากกว่าแค่ “ใครมีสิทธิ์ขาย OpenAI”

สำหรับคนทำระบบ นี่แปลว่าคำถามใหม่ไม่ใช่แค่ “ใช้โมเดลอะไร” แต่รวมถึง “จะ deploy บน cloud ไหนเพื่อให้ได้ cost, latency, data governance และ region ที่เหมาะที่สุด”

## Microsoft เสียอะไร และได้อะไร

จากมุมมองเร็ว ๆ คนอาจมองว่า Microsoft เสียสิทธิ์ผูกขาด

แต่มองให้ครบกว่านั้น Microsoft ยังได้ของสำคัญอยู่มาก:

- ยังเป็น **primary cloud partner**
- ยังมี license ถึง **2032**
- ยังมีส่วนร่วมทางการเงินใน OpenAI
- ยังได้รับ revenue share จาก OpenAI ต่อถึง **2030**

สิ่งที่ Microsoft ยอมปล่อยคือ “ความ exclusive” บางส่วน เพื่อแลกกับสัญญาที่นิ่งขึ้นและคาดการณ์ได้ง่ายขึ้น

ในโลกจริง นี่อาจมีค่ามากกว่าสิทธิ์ที่ดูแข็งแต่บังคับใช้ยาก เพราะความต้องการ compute ของ OpenAI โตเร็วจนการมีทางเลือกเพิ่มฝั่ง capacity กลายเป็นเรื่องจำเป็น ไม่ใช่แค่เรื่องต่อรองธุรกิจ

## OpenAI ได้อะไรจากดีลนี้

ฝั่ง OpenAI ได้ประโยชน์ชัดที่สุดในเรื่อง **อิสระในการขยายตัว**

เมื่อความต้องการ inference และ training โตระดับต้องพูดถึง **gigawatts ของดาต้าเซ็นเตอร์** การผูกการเติบโตไว้กับผู้ให้บริการรายเดียวเริ่มเป็นความเสี่ยงมากกว่าข้อได้เปรียบ

ข้อตกลงใหม่ทำให้ OpenAI:

- เปิดทางไปสู่ multi-cloud ได้จริง
- ลดข้อจำกัดด้าน capacity
- คุยกับพาร์ตเนอร์ใหม่ได้ง่ายขึ้น
- ทำแผน commercial scale-out ได้ยืดหยุ่นกว่าเดิม

สำนักข่าวหลายแห่งในวันที่ **28 เมษายน 2026** จึงตีความตรงกันว่าดีลนี้เป็นสัญญาณว่า OpenAI กำลังขยับจากความสัมพันธ์แบบ “พันธมิตรผูกแน่น” ไปสู่การเป็นบริษัทแพลตฟอร์ม AI ที่พร้อมวางบริการบนหลายโครงสร้างพื้นฐาน

## สิ่งที่ลูกค้าองค์กรควรอ่านจากข่าวนี้

สำหรับองค์กร ข่าวนี้สำคัญกว่าดราม่าระหว่างสองบริษัท

สิ่งที่ควรจับตาคือ:

### 1. โมเดลชั้นนำจะกระจายตัวมากขึ้น

ตลาดกำลังเดินไปทางที่โมเดลระดับท็อปไม่ได้ผูกกับ cloud เดียวแบบถาวรอีกแล้ว

### 2. การเลือก vendor จะซับซ้อนขึ้น

การตัดสินใจในปี 2026 ต้องดูพร้อมกันทั้ง:

- model quality
- region availability
- compliance
- cost
- throughput
- vendor lock-in risk

### 3. ความได้เปรียบจะย้ายไปอยู่ที่ execution

เมื่อโมเดลเข้าถึงได้กว้างขึ้น ความต่างจริงจะอยู่ที่ว่าใครเชื่อมมันเข้ากับ workflow, data, security และ production ops ได้ดีกว่า

## มุมมองของผม

ข่าวนี้สะท้อนสิ่งหนึ่งชัดมาก: **AI ปี 2026 โตเร็วจนสัญญาแบบยุคแรกเริ่มเริ่มไม่พอรองรับความจริงของตลาด**

Microsoft กับ OpenAI ไม่ได้ “เลิกกัน” และก็ไม่ได้ “กลับไปเหมือนเดิม” แต่กำลังเปลี่ยนความสัมพันธ์ให้สอดคล้องกับโลกที่:

- compute แพงและหายาก
- ดีมานด์ของลูกค้าโตเร็ว
- multi-cloud กลายเป็นเรื่องปฏิบัติจริง
- การแข่งขันไม่ได้อยู่แค่ที่โมเดล แต่อยู่ที่ช่องทางการให้บริการด้วย

ถ้าต้องสรุปสั้นที่สุด ข่าววันที่ **27 เมษายน 2026** คือวันที่ตลาดได้รับคำตอบว่า
**OpenAI ยังโตไปกับ Microsoft ได้ แต่จะไม่โตอยู่ใต้ Microsoft เพียงรายเดียวอีกต่อไป**
