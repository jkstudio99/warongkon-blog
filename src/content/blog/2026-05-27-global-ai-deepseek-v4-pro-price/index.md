ข่าว **Global / AI** สำหรับรอบวันที่ **27 พฤษภาคม 2026** คือการเปลี่ยนแปลงราคา API ของ **DeepSeek V4-Pro** ที่ตรวจสอบได้จากหน้า Models & Pricing ของ DeepSeek: บริษัทระบุว่าเมื่อโปรโมชั่นส่วนลด 75% สิ้นสุดในวันที่ **31 พฤษภาคม 2026 เวลา 15:59 UTC** ราคา API ของรุ่น V4-Pro จะถูกปรับอย่างเป็นทางการให้เหลือ **หนึ่งในสี่ของราคาเดิม**

Computerworld รายงานการปรับราคานี้เมื่อวันที่ **25 พฤษภาคม 2026** ขณะที่การตรวจหน้าเอกสารต้นทางในรอบวันที่ 27 พฤษภาคมพบข้อความเดียวกันอยู่ใน footnote ของตารางราคา นี่จึงไม่ใช่เพียงการคาดการณ์เรื่อง price war แต่เป็นข้อมูลต้นทุนที่นักพัฒนาและทีมจัดซื้อสามารถใช้วางแผนระยะใกล้ได้ โดยยังต้องตรวจราคาจริงอีกครั้งเมื่อเลยวันเปลี่ยนแปลง

## ราคาที่หน้าเอกสารแสดงอยู่

เอกสารของ DeepSeek แสดงราคาเป็นดอลลาร์สหรัฐต่อ **1 ล้าน tokens** สำหรับ `deepseek-v4-pro` ในช่วงที่มีส่วนลดดังนี้:

- input เมื่อ cache hit: **$0.003625** ลดจาก $0.0145
- input เมื่อ cache miss: **$0.435** ลดจาก $1.74
- output: **$0.87** ลดจาก $3.48
- concurrency limit ต่อ account: **500**

หน้าเดียวกันแสดงว่า V4-Pro รองรับ context length **1M** และ maximum output **384K** รวมถึงรองรับ JSON output, tool calls และทั้ง OpenAI-compatible กับ Anthropic-compatible base URLs

จุดที่ต้องอ่านให้ถูกคือส่วนลดที่เห็นในตารางยังถูกติดป้ายว่า 75% off จนถึงเวลาสิ้นสุดโปรโมชั่น แต่ footnote ระบุว่าหลังจากนั้นบริษัทจะปรับราคาอย่างเป็นทางการเป็นหนึ่งในสี่ของราคาเดิม ดังนั้นผลเชิงปฏิบัติคือราคาลดระดับนี้จะไม่จบลงพร้อม promo ตามข้อความปัจจุบันของเอกสาร

## ทำไมข่าวราคาจึงสำคัญกว่า benchmark เพียงอย่างเดียว

การเลือกโมเดลสำหรับ production ไม่ได้ตัดสินด้วยคะแนน benchmark เท่านั้น ทีมที่รัน agent, code assistance หรือ pipeline สรุปเอกสารจำนวนมากต้องรับต้นทุน output token, cache behavior, concurrency และความเสี่ยงจากการเปลี่ยนรุ่น API พร้อมกัน

การลดราคา V4-Pro มีผลอย่างน้อยสามด้าน:

- ทีมที่ทดลองโมเดลระดับใหญ่สามารถขยายจำนวน evaluation run ได้ในงบเท่าเดิม
- ผู้ให้บริการโมเดลรายอื่นถูกกดดันให้ชี้แจงทั้งราคาและคุณภาพของ reasoning/tool use ให้ชัดขึ้น
- การคำนวณต้นทุน agent ต้องให้น้ำหนักกับ cache hit และจำนวนรอบเรียกเครื่องมือ มากกว่าดูราคา input หนึ่งครั้ง

อย่างไรก็ตาม ราคาถูกลงไม่ได้รับประกันความเหมาะสมกับข้อมูลสำคัญขององค์กร ทีมยังต้องทดสอบ accuracy, latency, data-handling policy และ failure mode บน workload ของตนเอง

## การย้ายรุ่นมีรายละเอียดที่ห้ามมองข้าม

หน้าเอกสาร DeepSeek ระบุด้วยว่า model names เดิม `deepseek-chat` และ `deepseek-reasoner` จะถูกเลิกใช้ในอนาคต และในช่วง compatibility ปัจจุบันจะชี้ไปที่ non-thinking mode กับ thinking mode ของ **V4-Flash** ตามลำดับ ไม่ใช่ V4-Pro

รายละเอียดนี้มีผลต่อทีมที่คิดว่าการใช้ชื่อ endpoint เดิมจะได้รับคุณสมบัติหรือราคา V4-Pro โดยอัตโนมัติ การประเมิน production ควรระบุ model ID ชัดเจน บันทึกราคา ณ วันที่ทดสอบ และแยกผลของ Flash กับ Pro ออกจากกัน

## สิ่งที่ควรตรวจหลังวันที่ 31 พฤษภาคม

เมื่อถึงเวลาปรับราคาจริง ประเด็นที่ควรตรวจต่อมีดังนี้:

- หน้า pricing เปลี่ยนจากราคาโปรโมชั่นเป็น list price ใหม่ตาม footnote หรือไม่
- rate limit และ availability ของ V4-Pro เปลี่ยนตาม demand ที่เพิ่มขึ้นหรือไม่
- latency และคุณภาพ tool calls บน workload จริงเมื่อ traffic ขยายตัว
- กำหนดเลิกใช้ชื่อโมเดล compatibility เดิมถูกระบุเป็นวันที่แน่นอนเมื่อใด

## มุมมองของผม

ข่าวรอบนี้มีน้ำหนักเพราะเป็นการเปลี่ยนต้นทุนที่อ้างอิงจากหน้าเอกสารผู้ให้บริการโดยตรง ไม่ใช่การแข่งขันด้านข้อความโฆษณาเพียงอย่างเดียว สำหรับทีมที่ซื้อ AI API การลดราคาเป็นโอกาสให้ทดสอบกว้างขึ้น แต่ไม่ใช่เหตุผลให้ข้าม evaluation และ governance

สรุปสั้น ๆ:
**ณ วันที่ 27 พฤษภาคม 2026 เอกสารราคา DeepSeek ระบุว่า V4-Pro จะคงระดับราคาที่ลดลง 75% ผ่านการปรับราคาอย่างเป็นทางการหลังโปรโมชั่นสิ้นสุดวันที่ 31 พฤษภาคม ทีมใช้งานควรตรวจ list price และ performance ซ้ำหลังเวลามีผล**
